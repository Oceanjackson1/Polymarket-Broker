# Plan 3.5: CoinGlass Crypto Derivatives Data Integration — Design Spec

**Date**: 2026-03-19
**Status**: Draft
**Depends on**: Plan 3 (Data Pipeline — complete)

---

## 1. Overview

Integrate CoinGlass API v4 as the **second data pillar** alongside Polymarket, providing professional-grade crypto derivatives data (funding rates, open interest, liquidations, long/short ratios, taker volume) through a new `/api/v1/data/crypto/` route group. Add a BTC fusion endpoint that combines Polymarket prediction probabilities with CoinGlass derivatives indicators — a view unavailable anywhere else.

**Positioning**: LTP-style professional web trading platform for prediction markets. Data completeness and reliability > smart signals.

## 3. Data Model

### New table: `crypto_derivatives`

Append-only time-series table. One row per symbol per collect cycle.

```sql
CREATE TABLE crypto_derivatives (
    id              SERIAL PRIMARY KEY,
    symbol          VARCHAR(20)   NOT NULL,  -- BTC, ETH, SOL

    -- Funding Rate (aggregated across exchanges)
    funding_rate_avg    NUMERIC(12, 8),  -- OI-weighted average
    funding_rate_max    NUMERIC(12, 8),  -- highest exchange
    funding_rate_min    NUMERIC(12, 8),  -- lowest exchange
    funding_rates_json  JSONB,           -- full per-exchange breakdown
    next_funding_time   BIGINT,          -- unix ms

    -- Open Interest
    oi_total_usd        NUMERIC(20, 2),
    oi_change_pct_5m    NUMERIC(8, 4),
    oi_change_pct_15m   NUMERIC(8, 4),
    oi_change_pct_1h    NUMERIC(8, 4),
    oi_change_pct_4h    NUMERIC(8, 4),
    oi_change_pct_24h   NUMERIC(8, 4),
    oi_exchanges_json   JSONB,           -- per-exchange OI breakdown

    -- Liquidation (rolling windows)
    liq_long_1h_usd     NUMERIC(20, 2),
    liq_short_1h_usd    NUMERIC(20, 2),
    liq_long_4h_usd     NUMERIC(20, 2),
    liq_short_4h_usd    NUMERIC(20, 2),
    liq_long_24h_usd    NUMERIC(20, 2),
    liq_short_24h_usd   NUMERIC(20, 2),

    -- Taker Buy/Sell Volume
    taker_buy_ratio     NUMERIC(8, 4),   -- % (e.g. 53.95)
    taker_sell_ratio    NUMERIC(8, 4),
    taker_buy_vol_usd   NUMERIC(20, 2),
    taker_sell_vol_usd   NUMERIC(20, 2),

    -- Market Sentiment (daily, same value within a day)
    fear_greed_index    INTEGER,         -- 0-100

    recorded_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_crypto_deriv_symbol ON crypto_derivatives(symbol);
CREATE INDEX idx_crypto_deriv_recorded ON crypto_derivatives(recorded_at);
CREATE INDEX idx_crypto_deriv_sym_time ON crypto_derivatives(symbol, recorded_at DESC);
```

### Why separate table (not extending BtcSnapshot)?

1. **Multi-symbol**: BTC, ETH, SOL — BtcSnapshot is BTC-only by design
2. **Different data domain**: Derivatives data from centralized exchanges ≠ Polymarket prediction data
3. **Independent failure**: CoinGlass down → derivatives rows stop; CoinGecko+Polymarket data unaffected
4. **Different query patterns**: "Show me OI across all symbols" vs "Show me BTC prediction prob history"

### BTC Fusion: JOIN at query time

`/data/btc/fusion/{tf}` joins `btc_snapshots` + `crypto_derivatives` by matching `recorded_at` within ±30s window. No denormalization needed.

## 4. CoinGlassCollector

New collector, inherits `BaseCollector`. Runs independently from BtcCollector.

```
data_pipeline/coinglass_collector.py

- interval_seconds: 30
- symbols: ["BTC", "ETH", "SOL"]
- Per cycle: 4 API calls per symbol + 1 fear/greed = 13 calls total
- Graceful degradation: each API call try/except individually
  - If funding-rate fails → funding columns NULL, rest still collected
  - If entire cycle fails → BaseCollector catches, retries next cycle
```

### API calls per cycle:

| # | Endpoint | Params | Fields extracted |
|---|---|---|---|
| 1 | `funding-rate/exchange-list` | `symbol=X` | funding_rate per exchange, compute avg/min/max |
| 2 | `open-interest/exchange-list` | `symbol=X` | oi_total_usd, change_pct_*, per-exchange |
| 3 | `liquidation/coin-list` | (all coins) | filter by symbol, extract 1h/4h/24h long/short |
| 4 | `taker-buy-sell-volume/exchange-list` | `symbol=X&range=1h` | buy_ratio, sell_ratio, volumes |
| 5 | `fear-greed-history` | `limit=1` | latest index value (shared, called once) |

Call #3 and #5 return all symbols at once → called once per cycle, not per symbol.
Actual calls: 2 shared + 2 per symbol × 3 symbols = **8 calls/cycle** = 16 req/min. Within 80 req/min limit.

## 5. API Endpoints

All under `/api/v1/data/crypto/`, requiring `X-API-Key` + `data:read` scope.

### 5.1 Funding Rates
```
GET /api/v1/data/crypto/funding-rates?symbol=BTC
  → Latest funding rate per exchange + aggregated avg
  → Response: { symbol, aggregated: {avg, min, max}, exchanges: [{name, rate, interval, next_time}], stale, recorded_at }

GET /api/v1/data/crypto/funding-rates/history?symbol=BTC&limit=100
  → Time-series of aggregated funding rate
  → Response: { data: [{funding_rate_avg, recorded_at}], pagination }
```

### 5.2 Open Interest
```
GET /api/v1/data/crypto/open-interest?symbol=BTC
  → Latest OI + change % across timeframes + per-exchange breakdown
  → Response: { symbol, total_usd, changes: {5m, 15m, 1h, 4h, 24h}, exchanges: [...], stale, recorded_at }

GET /api/v1/data/crypto/open-interest/history?symbol=BTC&limit=100
  → Time-series of total OI
  → Response: { data: [{oi_total_usd, oi_change_pct_1h, recorded_at}], pagination }
```

### 5.3 Liquidations
```
GET /api/v1/data/crypto/liquidations?symbol=BTC
  → Latest liquidation stats (1h/4h/24h, long vs short)
  → Response: { symbol, windows: {1h: {long_usd, short_usd}, 4h: {...}, 24h: {...}}, stale, recorded_at }
```

### 5.4 Taker Volume
```
GET /api/v1/data/crypto/taker-volume?symbol=BTC
  → Latest taker buy/sell ratio + volume
  → Response: { symbol, buy_ratio, sell_ratio, buy_vol_usd, sell_vol_usd, stale, recorded_at }
```

### 5.5 Market Sentiment
```
GET /api/v1/data/crypto/sentiment
  → Fear & Greed index + latest derivatives summary for BTC
  → Response: { fear_greed: {value, label}, recorded_at }
```

### 5.6 Overview (Dashboard Aggregate)
```
GET /api/v1/data/crypto/overview?symbol=BTC
  → All 4 indicator categories in one call
  → Response: { symbol, funding: {...}, open_interest: {...}, liquidations: {...}, taker_volume: {...}, fear_greed: int, stale, recorded_at }
```

### 5.7 BTC Fusion (Polymarket × CoinGlass)
```
GET /api/v1/data/btc/fusion/{timeframe}
  → Polymarket prediction prob + CoinGlass derivatives for same time window
  → Response: {
      timeframe,
      polymarket: {up_prob, volume, market_id},
      spot: {price_usd},
      derivatives: {funding_rate_avg, long_short_ratio, oi_total_usd, oi_change_pct_1h, taker_buy_ratio, liq_long_1h, liq_short_1h},
      stale, recorded_at
    }
```

**Total: 8 endpoints** (6 crypto + 1 crypto overview + 1 btc fusion)
