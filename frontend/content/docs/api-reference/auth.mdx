# Authentication

Base path: `/api/v1/auth`

Authenticate via Google OAuth or wallet signature (EIP-4361). Once authenticated, manage your user profile and create scoped API keys for programmatic access.

## Endpoints

| Method | Path | Description | Auth |
|---|---|---|---|
| `POST` | `/auth/google` | Google OAuth login | -- |
| `POST` | `/auth/wallet/challenge` | Get nonce for wallet signing | -- |
| `POST` | `/auth/wallet/verify` | Verify wallet signature, receive JWT | -- |
| `GET` | `/auth/me` | Get current user profile | JWT |
| `PATCH` | `/auth/me` | Update display name | JWT |
| `GET` | `/auth/keys` | List all API keys | JWT |
| `POST` | `/auth/keys` | Create a new API key | JWT |
| `DELETE` | `/auth/keys/{key_id}` | Delete an API key | JWT |

---

## POST /auth/google

Authenticate using a Google OAuth credential token (from Google Sign-In). Returns a JWT access token and refresh token.

**Request body:**

```json
{
  "credential": "eyJhbGciOiJSUzI1NiIsInR5cCI6..."
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `credential` | string | Yes | Google OAuth ID token from client-side sign-in |

**Example request:**

```bash
curl -X POST https://api.polydesk.io/api/v1/auth/google \
  -H "Content-Type: application/json" \
  -d '{"credential": "eyJhbGciOiJSUzI1NiIsInR5cCI6..."}'
```

**Example response:**

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiJ9...",
  "refresh_token": "rt_abc123...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "user": {
    "id": "usr_abc123",
    "email": "user@example.com",
    "display_name": "Trader One",
    "avatar_url": "https://lh3.googleusercontent.com/a/...",
    "created_at": "2026-01-15T10:00:00Z"
  }
}
```

## POST /auth/wallet/challenge

Request a nonce for wallet-based authentication (EIP-4361 Sign-In with Ethereum). The client signs this nonce with their wallet and submits it to `/auth/wallet/verify`.

**Request body:**

```json
{
  "wallet_address": "0x1234567890abcdef1234567890abcdef12345678"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `wallet_address` | string | Yes | Ethereum wallet address |

**Example request:**

```bash
curl -X POST https://api.polydesk.io/api/v1/auth/wallet/challenge \
  -H "Content-Type: application/json" \
  -d '{"wallet_address": "0x1234567890abcdef1234567890abcdef12345678"}'
```

**Example response:**

```json
{
  "nonce": "a1b2c3d4e5f6...",
  "message": "Sign this message to authenticate with Polydesk.\n\nNonce: a1b2c3d4e5f6...\nIssued At: 2026-03-31T12:00:00Z\nExpires: 2026-03-31T12:05:00Z",
  "expires_at": "2026-03-31T12:05:00Z"
}
```

## POST /auth/wallet/verify

Submit the signed nonce to complete wallet authentication and receive a JWT.

**Request body:**

```json
{
  "wallet_address": "0x1234567890abcdef1234567890abcdef12345678",
  "signature": "0xabcdef..."
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `wallet_address` | string | Yes | Ethereum wallet address (must match the challenge request) |
| `signature` | string | Yes | EIP-191 personal signature of the challenge message |

**Example request:**

```bash
curl -X POST https://api.polydesk.io/api/v1/auth/wallet/verify \
  -H "Content-Type: application/json" \
  -d '{
    "wallet_address": "0x1234567890abcdef1234567890abcdef12345678",
    "signature": "0xabcdef1234567890..."
  }'
```

**Example response:**

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiJ9...",
  "refresh_token": "rt_wallet_xyz...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "user": {
    "id": "usr_def456",
    "wallet_address": "0x1234567890abcdef1234567890abcdef12345678",
    "display_name": null,
    "created_at": "2026-03-31T12:00:05Z"
  }
}
```

**Request body:**

}

| Field | Type | Required | Description |

  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiJ9..." \

```json
{

  "display_name": "AlphaTrader",
  "updated_at": "2026-03-31T12:00:00Z"
}
```

## POST /auth/keys

Create a new API key with specified scopes.

**Request body:**

```json
{
  "name": "Trading Bot v2",
  "scopes": ["read", "trade"]
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | Yes | Human-readable name for the key |
| `scopes` | string[] | Yes | Permission scopes: `read`, `trade`, `admin` |

**Example request:**

```bash
curl -X POST https://api.polydesk.io/api/v1/auth/keys \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiJ9..." \
  -H "Content-Type: application/json" \
  -d '{"name": "Trading Bot v2", "scopes": ["read", "trade"]}'
```

**Example response:**

```json
{
  "key_id": "key_ghi789",
  "name": "Trading Bot v2",
  "api_key": "pk_live_ghi789_full_secret_shown_once",
  "scopes": ["read", "trade"],
  "created_at": "2026-03-31T12:00:00Z"
}
```

> **Important:** The full `api_key` value is only returned once at creation time. Store it securely. It cannot be retrieved again.

## DELETE /auth/keys/\{key_id\}

Permanently deletes an API key. Any requests using this key will immediately fail.

**Path parameters:**

| Param | Type | Description |
|---|---|---|
| `key_id` | string | API key identifier |

**Example request:**

```bash
curl -X DELETE https://api.polydesk.io/api/v1/auth/keys/key_def456 \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiJ9..."
```

**Example response:**

```json
{
  "key_id": "key_def456",
  "deleted": true
}
```
